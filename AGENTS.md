# AGENTS

- `/docs` 下所有文档都要长期维护。
- 每一个修改都必须提交 Git，并提升一个小版本号（patch）。
- 每次改动都必须评估是否需要同步维护相关文档；需要时当次一并更新。
- 不做“最小修改”式修补：修改要为长期可维护性、可扩展性、代码清晰度与质量负责。
- 如果用户要求 push 到 GitHub，记得同步更新 GitHub Releases。
- 更新 Releases 时，默认写上非开发者推荐更新方法：
  - 去 Releases 下载 `dist.zip`
  - 先解压
  - 保持原来的 `AI-Shortcuts-dist` 路径不变
  - 不要在 Chrome 里“移除扩展”
  - 清空旧文件后放入新版内容
  - 回到 `chrome://extensions` 点当前实例的「重新加载」
- 如果用户明确要求写更新日志，再额外写“更新了什么”；如果用户没要求，就不要写更新日志。
